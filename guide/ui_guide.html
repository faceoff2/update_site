1. 개요

마크업 개발은 웹 개발의 기본 골격이며, 디자인과 개발의 중심에서 웹사이트를 단단하게 만들어주는 역할은 한다.
단단한 웹사이트, 댄시더홈이 제안한 방탄조끼 같은 웹사이트를 만들기 위해서는 최적화된 코드를 작업자간의 약속으로 지켜야 한다. 이 가이드 문서는 최적화된 코드를 가질 수 있도록 표준을 제시한다.

프로젝트 작업을 진행함에 있어서 공동작업에 효율적으로 대처하기 위하여 표준화된 코딩규칙을 제시하고, 공유하여 코딩규칙이 통일된 파일을 만들고자 함에 목적이 있으며, 추후 수정, 보완이나 유지보수 시 용이하도록 하기 위하여 통일된 형식으로 코드를 작성하여 업무 효율을 높일 수 있도록 한다.

수주 프로젝트 특성상 클라이언트의 요구되는 사항이 있을 시는 본 내용을 따르지 않을 수 있다.

2. 기본 규칙

  1. HTML의 문서는 의미 있는 마크업, 논리적인 구조로 구성되어야 한다.
  2. 레이아웃 설계 시 컨텐츠가 없을 경우에는 Footer가 항상 브라우저 하단에 붙게 한다.

    * 기본적인 레이아웃인 경우만 해당, 브라우저의 세로 스크롤이 나와도 상관 없는 디자인 레이아웃인 경우에는 제외

  3. 문서는 반드시 DTD를 선언하도록 한다. HTML5와 XHTML 타입의 DTD를 프로젝트의 성격에 맞게 사용하며, XHTML 문서의 경우는 DTD는 Transitional 타입을 사용한다.
  4. W3C의 Validator(한글버전, http://validator.kldp.org) 를 통해 최소한의 디버깅을 거칠 것.

3. HTML 문법

  1. 정렬간격

    Editplus 및 기타 에디터 기본 간격 설정은 설정 > 설정 및 구문강조 > 탭/들여쓰기 항목에서 설정한다.

    탭 : 4 / 들여쓰기 : 4 / 탭 대신 공백삽입에 체크 해제
  2. 대/소문자의 사용

    모든 태그 소문자로 작성한다. (id와 class값 및 css 포함, DTD 제외, js 제외)

  3. 닫힘 태그의 사용

    홀로 쓰이는 태그는 닫힘 태그를 사용한다 <input /> <br /> <hr /> <col /> <img />

    * HTML5에서는 닫힘 태그를 사용하지 않아도 무방하다, 하지만 웹 사이트의 통일성을 갖출 것

  4. 주석처리

    작업자간의 원활한 커뮤니케이션을 위하여 해당 요소 별로 구분되어 질 수 있는 주석을 표시한다.

    <!–- S: 해당 태그 네임 -->    시작 주석
    .
    .
    <!–- // E: 해당 태그 네임 -->    마무리 주석

    <!– 해당 태그 네임 ->     한 줄 주석 일 경우

    <!–------- 해당 태그 네임 ---------->
    이러한 방식의 주석은 사용하지 않도록 한다. 두개 이상의 하이푼 사용 금지(X)
  5. 공백의 빈 줄 지양

    HTML코드의 빈 줄은 의미 있는 객체를 구분하기 위하여 사용한다. 빈 줄은 한 줄로 제한한다.

    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    <div class="toparea"></div>
    빈 줄
    빈 줄
    <div class="leftarea"></div>     		<div class="toparea"></div>
    빈 줄
    <div class="leftarea"></div>
  6. 블록 태그와 인라인 태그

    태그 사용시 줄바꿈이 되는지의 여부에 따라 블록 태그와 인라인 태그로 나눌 수 있다. 또한 인라인 태그는 좌우에만 여백을 줄 수 있으며, 상하로는 여백을 줄 수 없다는 차이점이 있다.

    중간단계인 inline-block 태그를 필요에 따라 사용 할 수 있다.

    대표 block태그 : <div>, <p>, <ul>, <li>, <dl>, <dt>, <dd>, <form> 등
    대표 inline태그 : <span>, <strong>, <em>, <i>, <img>, <a>, <input> 등
    inline-block 태그의 예시 : display:inline-block 속성으로 사용 가능하다
  7. 모든 속성은 큰 따옴표(" ")로 묶는다 (홑 따옴표 사용 불가)
    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    <img src='bg_box.gif' width='200' height='100' alt='' />

    <img src=bg_box.gif width=200 height=100 alt= />     		<img src="bg_box.gif" width="200" height="100" alt="" />
  8. 속성과 속성 값은 함께 사용되어야 한다
    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    <select selected><option>선택</option></select>

    <input type="checkbox" checked>     		<select selected="selected"><option>선택</option></select>

    <input type="checkbox" checked="checked">
  9. 엔티티코드의 사용

    특수 문자는 엔티티 코드로 변환하여 넣어준다. (링크 url의 특수문자 포함)
    엔티티 코드 참조 URL http://entitycode.com/

    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    <p>대답이 돌아왔다 "그대가 원하는 대로"</p>

    <a href="list.asp&id=2">     		<p>대답이 돌아왔다 ＆lt;그대가 원하는 대로＆gt;</p>

    <a href="list.asp＆id=2">
  10. 비표준 태그 사용 금지

    font, center, font, strike, u, s, applet, basefont, dir, isindex, xmp

  11. Image 태그에 width, height 속성 값은 쓰지 않는다. 썸네일에만 이미지 사이즈 속성은 선언한다. ex) <img src="" alt ="" />
    * 이미지 링크 시 생기는 border 는 css에서 제어한다, img {border:0}
  12. 인라인 스타일 사용 지양

    꼭 필요한 경우가 아니라면 태그 및 JS, CSS를 인라인 태그로 넣지 않는다. CSS로 처리 가능한 모든 스타일은 표현한다. (많은 양의 이미지 롤오버 함수 제외)

    가이드를 준수하지 않은 예시 - 사용 제한
    <div style="width:100px; height:50px">

    <table bgcolor="#ff0000">
  13. 웹문서의 인코딩 선언

    웹문서는 반드시 인코딩 정보를 선언하며, 문서 저장 시에도 선언한 인코딩 방식과 동일한 방식을 선택하여 저장한다. 클라이언트 및 개발파트와의 별도 협의가 없을 시에는 UTF-8로 선언 및 저장한다.

    DB저장 용량이나 트래픽에 영향을 많이 받을 경우엔 euc-kr로 선언하는 함을 권장하며, utf-8 은 한글을 3Byte, euc-kr 은 한글을 2Byte 차지하여 하위 브라우저 버전에서 링크의 스타일에 영향을 줄 수 있다.
    여러 방식의 인코딩 예시
    <meta charset="utf-8"> (HTML5, UTF-8, 영문, 중문, 일문, 한글 등 지원)

    <meta http-equiv="Content-Type" content="text/html; charset=utf-8">
    (XHTML1.0, UTF-8, 영문, 중문, 일문, 한글 등 지원)

    <meta charset="euc-kr"> (HTML5, EUC-KR, 한글만 지원)
    <meta charset="utf+8"> (HTML5, UTF+8, 베트남어 계열 지원)
  14. 각종 HTML 태그 요소의 속성
    1. <head>

      DTD, meta, title, style link, script link 순서로 요소를 선언한다.

    2. <body>

      필요 시에 각 메뉴 별로 body id 또는 class를 지정한다.

    3. <fieldset>

      폼 요소의 집합으로서 각 폼 요소들을 그루핑 하기 위해 선언한다.
      자식 요소인 <legend>에 그룹의 이름을 표현한다.

      <fieldset>
          <legend>검색</legend>
          <input type="text"><button type="button"></button>
      </fieldset>
    4. <input>, <label>

      text type일 경우 동일한 스타일의 텍스트박스 너비 값이 다르면 인라인 스타일로 width 값을 제어한다.

      <input type="text" style="width:200px">

      <label> 요소를 함께 선언하여, 접근성을 높이거나, <label> 을 사용하지 않을 경우에는 title 속성을 이용하여 접근 할 수 있다.

      <label for="telnum"><input type="text" id="telnum" style="width:200px">
      <input type="text" title="전화번호" style="width:200px">
    5. <select>

      동일한 스타일의 셀렉트박스 너비 값이 다르면 인라인 스타일로 width 값을 제어한다.

      <select style="width:200px"><option></option></select>
    6. <textarea>

      cols, rows 속성을 반드시 입력하도록 한다. (validation 오류)
      최소 cols ="30", rows="5" 의 사이즈 속성을 넣어주도록 한다.

      <textarea cols="" rows=""></textarea>
    7. <button>

      폼 전송 역할은 submit type을 사용하며, 열기/닫기, 접기/펼치기, 스와이프, 터치 등의 UI 제어시에는 button type으로 선언한다.

      <button type="button">펼쳐보기</button>
      <button type="submit">form 전송</button>
    8. <a>

      기본적으로 인라인 태그, CSS를 통해 블록 태그를 만들어 모바일 터치영역을 조절 하기도 하며, HTML5 부터는 a 태그 안에 블록 속성 태그를 포함 할 수 있다. (HTML5하위 DTD 에서는 불가)

    9. <iframe>

      페이지 성능에 영향을 주기 때문에, 사용을 자제하며, 부득이하게 사용해야 할 경우 width, height, title, frameborder, scrolling 속성을 선언한다.

    10. <img>

      src, width, height, title, alt 등의 속성을 순서대로 선택적으로 선언한다.

    11. <table>

      테이블은 게시판 및 데이터 테이블에서 사용한다. (들여쓰기 최소화 할 것)

      <table cellpadding="0" cellspacing="0" summary="">
      <caption></caption>
      <colgroup>
          <col width="100" />
          <col width="100" />
          <col width="*" />
      </colgroup>
      <thead>
      <tr>
          <th scope="cols">title1</th>
          <th scope="cols">title2</th>
          <th scope="cols">title3</th>
      </tr>
      </thead>
      <tfoot>
      <tr>
          <th>합계 등의 값이 들어갈 경우 사용</th>
          <td>data2</td>
          <td>data3</td>
      </tr>
      </tfoot>
      <tbody>
      <tr>
          <th>sub title1</th>
          <td>data2</td>
          <td>data3</td>
      </tr>
      <tr>
          <th>sub title2</th>
          <td>data2</td>
          <td>data3</td>
      </tr>
      </tbody>
      </table>

      <thead>, <tfoot>, scope 은 데이터만 있는 테이블 일 경우엔 생략 할 수 있다.

4. CSS 문법

  1. 소문자 작성 준수
  2. 속성 선언 방법

    인코딩 선언시만 큰따옴표 선언
    그 외의 경우는 작은 따옴표 선언 (url 및 filter, alpa, opacity 등)

    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    @charset 'utf-8';

    background:url(images/bg.gif) no-repeat

    filter:alpha(opacity="90");

    filter:progid:DXImageTransform.Microsoft.AlphaImageLoader(src="bg.png", sizingMethod="scale")     		@charset "utf-8";

    background:url('images/bg.gif') no-repeat

    filter:alpha(opacity='90');

    filter:progid:DXImageTransform.Microsoft.AlphaImageLoader(src='bg.png', sizingMethod='scale')
  3. CSS는 외부선언 방식을 택한다. <link rel="stylesheet" type="text/css" href="style.css" />
  4. id는 레이아웃 요소 또는 스킵 네비게이션(건너띄기) 링크 시에만 사용한다.
  5. 인코딩 통일

    HTML문서가 브라우저에서 바르게 해석 될 수 있도록 CSS의 인코딩은 HTML 인코딩과 동일하게 선언한다. CSS 상단에 선언한 인코딩과도 통일되어야 한다.

  6. 글꼴을 정하는 경우 font-family 선언 시 "돋움" 혹은 "굴림" 이라는 한글을 사용자 환경에서 인식하지 못하는 경우를 대비한다.
    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    font-family:굴림 font-family:gulim, '굴림'
  7. CSS 선언 시 사용빈도가 높은 순서위주로 선언 순서를 통일한다.
    ① display, visibility
    ② overflow , clear, float
    ③ position - top, right, bottom, left
    ④ width, height
    ⑤ margin, padding
    ⑥ border
    ⑦ background
    ⑧ color
    ⑨ vertical-align, text-align
    ⑩ font, font-weight, text-decoraiton
  8. 공백 및 빈 줄의 최소화

    CSS 선언 시 공백을 최소화 할 것, 하나의 속성이 끝날 때에만 띄어쓰기 공백을 넣는다. CSS코드의 빈 줄은 의미 있는 객체를 구분하기 위해 사용한다. 빈 줄은 한 줄을 초과하지 않는다.

    가이드를 준수하지 않은 예시 .panel {width:＾100px; padding:＾10px}
    .panel {＾width:＾100px; padding:＾10px＾}
    빈 줄
    빈 줄
    .boxstyle1 {width:100px; font-size:12px}
    가이드를 준수한 예시 .panel {width:100px; padding:10px}
    빈 줄
    .boxstyle1 {width:100px; font-size:12px}
  9. 수치 선언 시에는 0을 제외한 숫자에는 px등의 단위와 함께 사용한다.
    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    .panel {padding:10px 0px 5px 10px} .panel {padding:10px 0 5px 10px}
  10. CSS는 들여쓰기 하지 않는다. (미디어 쿼리, 프린트 설정 등의 중첩되는 중괄호는 제외)
    줄 바꿈은 클래스 또는 아이디를 새로 선언할 경우만 가능 (코드의 가독성을 위한 줄바꿈은 지양한다)
    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    #top {
            width:100px;
            height:90px;
    }
    ＾＾＾＾#top .gnb {
                    padding:0 16px;
                }     		#top {width:100px; height:90px}
    #top .gnb {padding:0 16px}

    @media all and (min-width:768px){
        body {width:100px; font-size:12px}
    }
  11. 외모만을 위한 중첩클래스는 사용은 최소화 한다.
    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    .clearfix {clear:both}
    .mt20 {margin-top:20px !important}
    .pd10 {padding-left:10px !important}
    .panel {width:100%; height:50px; font-size:12px}

    <div class="panel mt20 pl10">     		.panel {clear:both; width:100%; height:50px; padding-left:10px; margin-top:20px; font-size:12px}

    <div class="panel">
  12. 다중 선택자의 사용을 최소화 한다.

    (IE6에서 다중 선택자를 읽지 못하는 현상이 많다. IE6을 고려하지 않는 사이트 일 경우도 최소한으로만 사용할 것)

    가이드를 준수하지 않은 예시 가이드를 준수한 예시
    .panel {padding:10px 0px 5px 10px}
    .panel a.tab {display:block; text-align:center}
    .panel a.tab.on {background:#f00}     		.panel {padding:10px 0px 5px 10px}
    .panel a.tab {display:block; text-align:center}
    .panel a.on {background:#ff0000}
  13. 폼 태그에 해당하는 클래스는 아래를 사용한다.

    (20page 초기파일 예시에서도 확인 가능)

    input.txt {width:100px; height:17px; padding:3px 0 0 3px} /* 인풋 텍스트박스 전용, 속성 값은 변경 가능 */
    input.file {width:100px; height:17px; padding:3px 0 0 3px} /* 인풋 파일박스 전용, 속성 값은 변경 가능 */
    input.radio {vertical-align:middle} /* 인풋 라디오박스 전용 , 속성 값은 변경 가능 */

5. 이미지 및 파일명 네이밍, 폴더구조 규칙

  1. 공통 네이밍 규칙
    1. 영문 소문자와 숫자만 사용, 구분자는 "언더스코어( _ )" 를 사용한다.
    2. 클래스명 선언 시 "하이푼( - )" 을 사용할 수 있다.
    3. 숫자는 01, 02, 03… 으로 넘버링하며, 숫자는 맨 앞으로 올 수 없다.
    4. 직관적으로 어떤 목적인지 알 수 있는 단어를 사용한다.
    5. 대문자 소문자를 CamelCase 방식을 사용하지 않는다 (JS함수명 제외) ex) BtnClose (CamelCase 방식, X)   →   btn_close (O)
           NoticeList (CamelCase 방식, X)   →   notice_list (O)
